# async-llm

## 问题

当前所有主流大语言模型都是回合制的：用户说一句，模型回一句，严格交替。

但人类的真实交互几乎从不是回合制的——我们会连发消息、会停下来想、会沉默、会撤回、会被打断、会在对方还没说完时就开始组织回应。

我们想要一个不一样的模型：能像人一样**异步**地接收输入、**异步**地产生输出。

## 想要什么

不是"更快的回合制"，而是结构上不一样的交互范式。

### 沉默是合法输出

模型在每个时刻都在决策"现在该不该说话"。不说话不是缺席，是一种主动的、有意义的状态。回合制模型被强迫在每个 turn 都说点什么，这是失真的。

### 内部生成 与 外部表达 分开

模型应该有"想说但没说"的能力——在内部持续生成候选内容（草稿），但只有满足条件时才提交到对话流。

这对应人在微信里打字的过程：你可以打到一半撤回、删掉重打、想了想没发——只有"发送"按钮按下后内容才存在于对话里。说出口的话只是内部连续语言流里被"提交"的小部分。

### 撤回是基本操作

模型应该能"试探性生成 → 发现不对 → 丢弃 → 重来"。

这给了模型任意长的内部计算空间，而外部观察者只看到收敛后的输出。这正是图灵机的工作方式：纸带上的中间步骤对外不可见，只有最终结果重要。

### 时间维度可以被抽离

当模型的行为由"状态 + 输入 + 转移"三件事定义，而不是由物理时间定义，它就退化为一个纯粹的状态机。

这种形态下，模型不再是"在时间里运行的函数调用"，而是**持续存在的有状态实体**——可以挂起、可以恢复、可以累积经验、可以被识别为同一个个体。物理时间是实现细节，不是模型本体的一部分。

### 多消息、多线程、打断

模型应该能：
- 连发多条消息（"等下""哦对了""刚才说错了"）。
- 在多个话题间切换。
- 在被打断后自然衔接，把没说出口的话咽回去。
- 在长时间沉默后合理地恢复对话。

这些都是人类对话的常态，当前 chatbot 范式做不到。

## 这意味着什么

如果上面几点都做到了，得到的不只是"一个能异步聊天的 AI"，而是范式上不同的对象。

**从函数到机器**：当前 LLM 是"输入 prompt → 输出 token"的纯函数，每次推理都从 prompt 重读一遍历史，两次推理之间模型本身是无状态的。我们想要的是有内部状态、能持续演化的机器——它的"心智"是一个可以演化、可以被持久化的实体，不是一个无状态的服务调用。

**从有限自动机到图灵完备形态**：纯回合制 Transformer 在固定上下文下能力是有上界的。加上无限运行循环 + 草稿/撤回操作（即纸带的擦写能力），架构上就跨越到图灵完备的形态。这不是说模型在实践上能解决任何问题，而是说它脱离了"看似强大但本质是有限自动机"的状态。

**从助手到伙伴**：能"在场"而不是每次被召唤；能保持沉默也能主动开口；能记得自己刚才想说什么但没说出口。这是从"问答助手"到"持续在场的对话伙伴"的产品形态转变。

## 最终目标

我们最终希望这样的模型能跑在**端侧**——个人设备上持续存在的、有状态的、异步的 AI 伙伴。

这是为什么状态压缩、持久化、低算力需求会是后期的关键约束。但这一阶段只描述想要什么，实现路径暂不展开。

## 不在范围内

- **把回合制对话做得更快**：这是工程优化，不是范式转变。
- **多模态 / 全双工语音**：聚焦文本，语音引入完全不同的数据问题（ASR、说话人分离、韵律），证明文本异步可行后再扩展。
- **具体训练方法、架构选型、数据 pipeline**：之后再讨论。这份文档只描述目标，不规定路径。
